Como Empacotar Script Python em um .exe

Guia completo para criar executáveis que não abrem a janela do console e com ícone personalizado.

Introdução

A maneira mais fácil e popular de transformar seus scripts Python em executáveis standalone (`.exe`) para Windows é usando o PyInstaller. Ele empacota tudo o que seu aplicativo precisa para funcionar em um único distribuível.

O segredo para não abrir a janela do console (CMD) é usar uma opção específica que informa ao PyInstaller que sua aplicação é uma "aplicação de janela" (windowed application). Este tutorial também mostrará como personalizar o ícone do seu executável.

Passo 1: Instalar o PyInstaller

Se você ainda não o tem, abra seu terminal ou Prompt de Comando e instale-o usando o pip:

pip install pyinstaller

Passo 2: O Comando Mágico (`--noconsole`)

Depois de instalar, navegue até a pasta onde está o seu script Python (`.py`) e use o seguinte comando:

pyinstaller --noconsole seu_script.py

O que esse comando faz?

  • pyinstaller: Invoca a ferramenta.
  • --noconsole: Esta é a opção chave! Ela diz ao PyInstaller para criar um executável que não abre uma janela de console. Um atalho para ela é -w (de "windowed"). Então, pyinstaller -w seu_script.py faz a mesma coisa.
  • seu_script.py: Substitua pelo nome do seu arquivo Python.

Passo 3: Entendendo o Resultado

Após executar o comando, o PyInstaller criará algumas pastas:

  • build/: Contém arquivos temporários. Você pode ignorar ou deletar.
  • dist/: É aqui que está o seu ouro! Dentro da pasta dist, haverá uma outra pasta com o nome do seu script. Dentro dela, você encontrará o arquivo seu_script.exe e todas as suas dependências.
  • seu_script.spec: Um arquivo de configuração para personalizações avançadas.

Para distribuir seu programa, você só precisa enviar a pasta inteira que está dentro de dist/.

Exemplo Prático: Criando um App com Interface Gráfica

Vamos criar um script simples usando tkinter para demonstrar.

1. Crie o arquivo app.py:

import tkinter as tk
from tkinter import messagebox

def mostrar_mensagem():
    messagebox.showinfo("Olá!", "Este é um aplicativo GUI sem janela de console!")

# Cria a janela principal
janela = tk.Tk()
janela.title("Meu App")
janela.geometry("300x150")

# Cria um botão
botao = tk.Button(janela, text="Clique em mim!", command=mostrar_mensagem)
botao.pack(pady=20, padx=20)

# Inicia o loop da aplicação
janela.mainloop()

2. Empacote com o PyInstaller:

pyinstaller --noconsole app.py

3. Execute o .exe:

Vá para a pasta dist/app/ e clique duas vezes em app.exe. Uma pequena janela gráfica aparecerá, mas nenhuma janela de CMD preta será aberta.

Personalizando o Ícone do Executável

Para dar um toque profissional, você pode definir um ícone personalizado para o seu .exe. O ícone precisa estar no formato .ico.

Use a flag --icon no comando:

pyinstaller --noconsole --icon=meu_icone.ico seu_script.py

Dica: Se o ícone não aparecer no Windows Explorer, pode ser um problema de cache. Reinicie o processo "Windows Explorer" no Gerenciador de Tarefas para forçar a atualização.

Opções Adicionais e Comando "Master"

Você pode combinar as opções para um resultado mais profissional.

  • Criar um Único Arquivo Executável (`--onefile`): Gera apenas um .exe em vez de uma pasta cheia de arquivos. 
    pyinstaller --noconsole --onefile seu_script.py

Um comando que combina as opções mais comuns é:

pyinstaller -w -F --icon=icone.ico --add-data "dados;dados" seu_script.py
  • -w: --noconsole (sem janela de console)
  • -F: --onefile (arquivo único)
  • --icon: Define o ícone
  • --add-data: Adiciona arquivos externos (imagens, configs, etc.). Formato: "origem;destino".

Alternativa: Usando `cx_Freeze`

O cx_Freeze é outra excelente alternativa. O processo é um pouco diferente, exigindo um arquivo setup.py.

1. Instale:

pip install cx_freeze

2. Crie o arquivo setup.py:

from cx_Freeze import setup, Executable

# "base" é o equivalente ao --noconsole do PyInstaller
# "Win32GUI" para aplicações GUI no Windows
base = "Win32GUI" 

setup(
    name = "Meu App GUI",
    version = "0.1",
    description = "Um aplicativo de exemplo",
    executables = [Executable("app.py", base=base, icon="meu_icone.ico")]
)

3. Execute o build:

python setup.py build

Resumo

Para a sua necessidade, o PyInstaller é a ferramenta mais direta. Lembre-se sempre de usar a flag --noconsole (ou -w) para garantir que a janela do CMD não apareça e --icon para personalizar o visual do seu aplicativo.

Ferramenta Como não abrir o console Como personalizar o ícone Formato do Ícone
PyInstaller Usar a flag --noconsole ou -w Usar a flag --icon=arquivo.ico .ico
cx_Freeze Usar base="Win32GUI" no Executable Adicionar icon="arquivo.ico" ao Executable .ico
